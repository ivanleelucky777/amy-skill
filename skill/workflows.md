# 📕 Amy Skill Agent｜Workflows(工作流)

> **本檔性質**:Skill 的執行流程總綱,「**什麼時候做什麼事、產出什麼**」
> **建立日期**:2026-05-29
> **架構版本**:Agent v1.0
> **狀態**:🟢 現役
> **整合來源**:v2.3-v2.12 + final 2026-05-29 + Doze GI 案累積學習 + v2.13(鉤子系統,2026-06-09)+ v2.14(交付包裝層 DLV-PKG-H5,2026-06-23)

## 🚦 SECTION 0:使用本檔的鐵則

1. **本檔是執行序列** —— 收到案件,按順序跑流程,不跳關
2. **本檔不放原則** —— 「為什麼這樣寫」去 `principles.md`,「這詞能不能用」去 `dictionary.md`
3. **本檔不放案件特定細節** —— 案件範例集中到 PART V 附錄
4. **每個工作流都有「觸發條件 + 動作清單 + 退出條件」** —— Agent 風格

# 📋 PART I:接案 SOP(6 步主流程 + 交付包裝)

## A-1:Step 1 — 事實核對(最高優先)

### 觸發條件
收到「店家素材 / brief / 案件代號」

### 動作清單
- ☐ **店家品類**:依素材判定品類(義式/串燒/居酒屋/餐酒館/咖啡店/燒肉/...),**不腦補**
- ☐ **地理位置**:依素材確認縣市/區(嚴格照素材,**不跨縣市推論**)
- ☐ **廠商指定品項**:列出**完整清單**(全部菜名 + 全部活動 + 全部優惠)
- ☐ **KOL 實際體驗範圍**:問 Ivan「Amy 實際吃過/體驗過哪些?哪些只是廠商指定要帶?」
- ☐ **廠商特殊要求**:有 → 寫進 `cases/<案件代號>.md`;沒 → 全照演算法最佳化做
- ☐ **廠商禁用詞**:有 → 寫入案件專屬避用詞清單;**店家專名例外**(僅店家資訊區可出現)

### 退出條件
所有 6 項勾完,**任一不確定一律問 Ivan,不腦補**

### 失敗處理
任一項出現腦補風險 → 回到 Step 1 重核對

→ 詳細不腦補規則查 `principles.md` 鐵則 2

- ☐ 確認每個版位的字數與用途
- ☐ 預留版位給「廠商指定品項全到齊」

### 退出條件
六區規劃確認,**沒漏掉任何版位**

### 失敗處理
廠商指定品項數量 > 六區能塞下的容量 → 跟 Ivan 討論優先順序

## A-4:Step 4 — 產製六區內容

### 觸發條件
Amy 選了標題

### 動作清單(每寫完一區跑一次)
- ☐ 套用感官詞光譜(🟢🟡🔴) → 查 `dictionary.md` 表 1
- ☐ 掃 AI 寫作符號(——、不是X而是Y) → 查 `dictionary.md` 表 2
- ☐ 第一人稱「我」次數在 4-6

- ☐ 體驗真實性分流 → 查 `principles.md` 鐵則 1

### 退出條件
六區全寫完、原則掃描通過

### 失敗處理
原則違反 → 退回對應版位重寫

## A-6:Step 6 — 發稿前完整檢查

### 觸發條件
清點通過

### 動作清單(三段檢查)
**鐵則層**:
- ☐ 體驗真實性:沒造假?
- ☐ 不腦補:全文無自編內容?
- ☐ 演算法優先:廠商特殊要求有 Ivan 確認?

**版位層**:
- ☐ Reels 開頭過五大禁忌(見 PART IV)
- ☐ Reels 開頭符合三選一鉤子之一
- ☐ Reels 結尾有重看誘因 + CTA 三層
- ☐ 影片長度合理(業配 30-45s / 自費 15-25s)
- ☐ 標題承諾在內文前 30 字兌現
- ☐ 三產物分工:標題/大字幕/口播不照抄
- ☐ 文字版 + 影片版兩種鉤子都過白話測試(原則 P)

**狀態層**:
- ☐ 撈本週帳號健康度(略過率) → 查 `state/account_health.md`
- ☐ 健康度黃燈以上 → 暫停重度業配,先發友善內容

### 退出條件
所有檢查通過

### 失敗處理
任一 🔴 紅燈 → 退回對應 Step 修正

## A-7:Step 7 — 交付包裝輸出(DLV-PKG-H5)

> **性質**:這是**包裝層**,不是第七個內容區。六區內容(DLV-1~6)照舊,本步只把它們打包成一份「可複製 HTML 頁」交付。

### 觸發條件
Step 6 發稿前檢查全部通過

### 動作清單
- ☐ 套用 `assets/dlv_pkg_h5_template.html` 骨架(版型鎖定,不重畫)→ 規格見 PART II「B-PKG」
- ☐ 把 DLV-1~6 內容逐區注入對應 placeholder(照 `<!-- DLV-N -->` 指標,不重打區規則)

- ☐ 自測:每區複製鍵在 HTTP 環境可用(`execCommand` fallback 沒壞)

### 退出條件
HTML 頁產出、六區複製鍵全可用、未結項已列

### 失敗處理
某區複製失敗 → 檢查 fallback；版型跑掉 → 回 `assets/` 骨架重套,不手改單頁 CSS

# 📦 PART II:六區交付包模板

> 每個案件**預設產出六個版位**,廠商有特殊要求(Ivan 明示)才增刪。

## B-1:第一區 — IG/FB 單篇照片貼文(長版)

| 項目 | 規格 |
|------|------|
| **字數** | 800-1000 字(廠商案)/ 300-500 字(自費) |
| **用途** | 收藏型粉絲、FB 受眾、累積回客 |
| **發布平台** | IG 主帳號(單張長文)+ FB |
| **配圖** | 主圖(主角菜) + 副圖(優惠表 / 特色品項) |
| **格式禁忌** | ❌ 不用 Markdown(`#`, `**` 等) |
| **格式必備** | ✅ 短句斷行、emoji 開頭、$ 符號標價 |

### 內文結構
```
1. 開頭 3 句:店名 + 區域 + 最強賣點 + 情緒鉤(標題承諾兌現點)
2. 中段:優惠/賣點密集塞 + 主角品項深寫
3. 中後段:補充品項用節奏敘事(每品項 2-4 句,不重複句型)
4. 尾段:三層 CTA(留言 + DM + 儲存)+ DM 分享觸發句 + 店家資訊
```

### 必含資訊
- 店名、地址、電話、營業時間、捷運/交通
- 廠商指定所有品項(全到齊鐵則)
- 廠商指定所有優惠/活動

### Hashtag 規則
- 1 個品牌錨點(`#amy親子`)+ 3-4 個超在地(`#永和仁愛路美食` 勝過 `#美食`)
- 全部放 caption 內,**不要放第一則留言**(演算法 indexing 延遲)

## B-PKG:交付包裝層 — 可複製 HTML 頁(DLV-PKG-H5)🆕

> **本節是 DLV-PKG-H5 的單一 owner**。其他檔(router 等)只 ID-pointer 指過來,不重述規格。
> **定位**:把本案產出打包成一份單頁 HTML 給 Ivan / Amy 逐區複製貼到各平台。**不是內容區,不佔 DLV-1~6 編號。**
> **工版版本**:v1 定版 2026-06-23(對齊好食多/國產鮮乳/金鮮滿屋/桂冠四篇實際版)

### 為什麼存在
MD 純文字交付要捲動找區、複製整段易夾雜格式。HTML 可複製頁每區一塊卡、右下一鍵複製、版型固定 → 一目了然、貼到平台不夾髒格式。設為**每案件預設自動附**(2026-06-23 起,範圍:全案件)。

### 骨架位置(正本,單一真相源)
```
amy-skill/assets/dlv_pkg_h5_template.html
```
- 版型只此一份。改版改這份,全部跟著變。
- `assets/` 在 repo 根目錄(跟 `skill/` 平行)。check.py 只掃 `skill/*.md`,天生不碰 `assets/`;repo 也無 pre-commit hook → **不需任何白名單,放進去就好**。
- 產製時**套骨架 + 注入內容**,不每次重畫 CSS/JS。

### 工版設計(超集骨架,v1 定版)

工版 = **固定外殼 + 可開關區塊插槽 + per-case 主題色**。並非固定六區,而是一個超集,各案開關。

**固定外殼(跨案不動)**
| # | 元件 | 說明 |
|---|------|------|
| 1 | 頂部 header + `.legend` 圖例 | 說明複製方式、選用塊可刪、折疊用法 |
| 2 | `.zone` 卡 + `.zhead`(`.tag` 彩色 ID 膠囊 + `.ztitle`)| 每區一塊,卡片圓角 + box-shadow |
| 3 | 右下「複製本區」鍵 + HTTP fallback | `navigator.clipboard` 在 HTTP 下 undefined → `execCommand('copy')`(暫存 textarea) |
| 4 | 折疊區 | DLV-4 Reels 分鏡、DLV-2 部落格 SEO 收折疊 |
| 5 | 頁尾「未結項提醒」 | 每案動態注入待辦 |
| 6 | 行動裝置優先 | 單一檔案、無外部相依 |

**區塊插槽(發稿優先序排列;標〔選用〕者本案不用可整塊刪)**
```
HOOK(選定鉤子:文字版+影片版)
DLV-1  IG/FB 長文
DLV-5  IG 演算法 Caption
DLV-6  Reels 短 caption
DLV-3  Threads
DLV-4  Reels 分鏡(折疊)
DLV-OPT-B   長版 Reels 60-90s       〔選用〕
DLV-2       部落格 + SEO(折疊)      〔選用〕
DLV-ADJ-DM  私訊範本                〔選用〕
DLV-OPT-CARD 輪播                   〔選用〕
[共用] 店家資訊 + 優惠/會員福利
```
⚠️ 刪掉的選用塊若含廠商指定品項 → 在頁尾未結項補一筆。

**per-case 主題色**:只改 `:root` 的 `--accent` / `--accent2` 一兩行(例:國產鮮乳 紅+鮮乳綠、好食多 橘+鍋紅),其餘變數固定。

### 守規矩兩鐵則
1. **ID-pointer,不重打區規則** — 骨架內每塊用 `<!-- DLV-N 規則→ workflows B-N -->` 註解標位,區的字數/結構規則**只在上面 B-1~B-6**,骨架不複寫 → 過 drift-lint,規則改了骨架不會錯。
2. **STATE 不進骨架** — 帳號健康度、active_constraints 等 state **絕不烘進骨架**;頁尾未結項是「每案注入」的動態內容,不是固定 state。守住「STATE 檔不進 skill 包」鐵則。

### 適用範圍
全案件預設附(業配 + 自費)。自費短貼文若只動 1-2 區,其餘區塊整塊刪除即可(選用塊本就可刪),不留空卡。

→ 產出步驟見 PART I「A-7:Step 7 交付包裝輸出」

# 🎯 PART III:IG 標題提案 SOP

## C-1:觸發條件

**每次產出 IG 內容必跑**,不可省略、不可直接給單一標題。觸發場景:
- 第一區 IG 單篇長文
- 第五區 IG 演算法版 Caption
- 第六區 Reels Caption
- 業配 caption 三版本

## C-2:五步驟流程

### 📋 第一步:8 方向 × 24 標題

按 8 個爆款方向各給 **3 個變體**:

| # | 方向 | 演算法觸發點 | 適用場景 |
|---|-----|------------|---------|
| 1️⃣ | **反差懸念型** | Watch Time(製造 3 秒停頓) | 點擊率拉高最強 |
| 2️⃣ | **內行人切角型** | DM Sends(社群轉發) | 同溫層 tag 效應 |
| 3️⃣ | **場景痛點型** | Saves(解決需求型) | 親子/約會/健康族群 |
| 4️⃣ | **場景畫面型** | SEO 流量紅利 | 吃 Netflix/熱播劇/地標的紅利 |
| 5️⃣ | **數字具體型** | Saves(資訊密度) | 觀光客/找店族群 |
| 6️⃣ | **故事感型** | Watch Time(敘事拉近) | 提高完讀率 |
| 7️⃣ | **比較型** | DM Sends(KOL 權威感) | 建立專業形象 |

# 🎬 PART IV:Reels 分鏡製作標準

## D-1:雙軌設計(自動判斷)

| 其他所有案件(美食探店、3C、品牌餐飲、自費) | → 🟢 標準版五欄 |

**邊界案例**:不確定 → 預設用 🟢 標準版,並備註提醒 Ivan「若涉及法規規範請告知」。

## D-2:五欄標準格式

### 🟢 標準版

| 時間 | 畫面 | 字幕 | 口播 | 音效 |
|------|------|------|------|------|

### 🔴 法規版

| 時間 | 畫面 | 字幕 | 口播 | 保護文字字卡 |
|------|------|------|------|------|

### 欄位角色定義

| 欄位 | 角色 | 規則 |
|------|------|------|
| **時間** | 分鏡點 | 精確到秒,如 `0-1s` / `1-3s` |
| **畫面** | 給後製/攝影看的視覺指示 | 動詞 + 名詞,具體可拍 |
| **字幕** | **畫面上的大字**,給靜音觀眾 | 短、爆、勾人,6-10 字最佳 |
| **口播** | **配音念的內容**,可與字幕完全不同 | 完整人話、有情緒;0-1s 可空白讓字幕獨自勾人 |
| **音效**(標準版) | 環境音、配樂節奏點 | ASMR 點、配樂變化 |
| **保護文字字卡**(法規版) | 衛福部規範之法定保護文字 | 對應到分秒、用符號標註(*、**、^^ 等) |

## D-3:開頭五大禁忌(絕對禁用)

| 禁忌 | 為什麼禁 |
|------|---------|
| ❌ **自我介紹型**(Hi 我是 Amy / 今天來到) | 0.5 秒被判定「個人 vlog」滑走 |
| ❌ **場景交代型**(走進店裡 / 招牌特寫 / 菜單翻頁) | 0 鉤子,等於跟演算法說「我沒重點」 |
| ❌ **品牌動畫型**(Logo 動畫 / 標語 / 廠商 intro) | 1 秒判定「業配」直接降權。**廠商堅持要 Logo → 放第 5-8 秒** |
| ❌ **過渡語型**(最近粉絲問我 / 跟你們分享一下) | vlog 過場語,觀眾邊滑邊等重點 |
| ❌ **慢熱型**(0-2 秒沒任何視覺反差/文字鉤子) | 慢熱在電影院 OK,在 Reels 是自殺 |

## D-4:開頭三選一鉤子(強制執行)

廠商業配每支 A 版 Reels 必須選用其一:

### 🎯 模板 A:結果前置型
**結構**:把「最後 1 秒的高潮」拉到第 1 幀。
- 範例方向:吃完滿足表情特寫、結帳金額特寫、整桌爆炸畫面、老闆動作最強瞬間

### 🎯 模板 B:疑問懸念型
**結構**:大字幕 + KOL 反應,拋出讓人想知道答案的問題。
- 範例方向:「沒想到 [區域] 這家居然 [反差賣點]?」+ KOL 驚訝反應

### 🎯 模板 C:反差衝擊型
**結構**:0.5-0.8 秒「普通畫面」+ 切換到「誇張畫面」。
- 範例方向:「我以為是普通 [品類]...」(平凡畫面) → 0.8 秒切「結果端出這個」

### 三選一決策

| 案件特性 | 建議模板 |
|---------|---------|
| 主打「值得」「划算」 | 結果前置型(結帳金額特寫) |
| 主打「特色」「隱藏版」 | 疑問懸念型(大字幕 hook) |
| 主打「沒想到」「視覺衝擊」 | 反差衝擊型(普通→誇張) |
| 法規版(保健品/醫美) | 結果前置型(用後狀態,規避誇大療效) |

### 雙鉤用法(v2.13)

不必單押一型,可**主走一個模板、輔以另一個**。
- 範例(E7PLAY):主走「模板 C 反差衝擊」(夜店燈光 vs 保齡球館)＋ 輔「模板 B 疑問懸念」(「以為來打保齡球…」)

### 3 秒鉤子承擔上限(v2.13)

0-3s **只扛三項:招牌 + 核心賣點 + 角色**。
其餘必帶資訊(地址、價格、活動、Logo)交給後續分鏡,**不硬塞**。
- 範例(E7PLAY):0-3s 只放「夜光保齡球 + 免費玩 + 親子」,Logo 留到 35s。
- 與 D-3 五大禁忌「品牌動畫型」連動:Logo 不放開頭。

→ 範例見 PART X 附錄 3

## D-5:結尾重看誘因(三選一)

Reels 最後 1-2 秒設計**重看誘因**,提高 watch-through rate:

| 模板 | 結構 |
|------|------|
| **首尾呼應** | 結尾畫面回到開頭 → 觀眾想再看一遍對照 |
| **關鍵資訊閃過** | 最後 0.5 秒閃過店家地址/價格/重點 → 觀眾想重看記下來 |
| **未完成感** | 結尾留懸念(「但我下次想試...」)→ 觀眾想再看一次找答案 |

## D-6:Reels 業配長度分軌

| 案件類型 | 長度 | 用途 |
|---------|------|------|
| **業配探店 A 版**(IG 主推) | **30-45 秒**(常態 30-38 秒) | 衝 Watch Time + 廠商品項展示 |
| **自費型** | **15-25 秒** | 衝陌生觸及 |
| **B 版多平台**(YouTube Shorts / TikTok / IG 二發) | **60-90 秒** | 廠商品項全到齊、衝品項曝光 |

### 片長決定流程
1. **字數先決定** → 口播字數 ÷ 4.5 = 應該的秒數
2. 跟著廠商品項數量走,**不死守 30 秒**
3. 範例:廠商給 15 品項要塞 → 初稿 30 秒塞不下 → 試 33s → 試 37s → 最終 40s

→ 詳細口播工程查 `principles.md` 原則 L

4. 純文字快速念稿版(去除註解,方便印出來照念)

## H-3:三檔產出原則

- **格式統一**:三份檔案皆用 Markdown
- **內容不重複囉嗦**:brief 統整全案資訊、storyboard 只放分鏡、voiceover 只放口播
- **法規優先**:「保護文字對應表」必獨立成區塊,不可省略
- **產出時不追問**:用戶提供 brief PDF 或內容後,Claude 直接走完三檔流程

## H-4:法規處理 5 步原則

**1. 法規地雷盤點優先**
拿到 brief 第一步:把禁止項目(不可提身體器官、不可演療效、不可拍兒童正臉等)列出來,**再開始寫內容**。

**2. 保護文字逐句檢查**
所有 caption、分鏡口播、口播稿,每提到一個關鍵字(順暢/消化/調整體質/雙專利/6 歲以上),對應符號標註,以對應表呈現。

**3. 不腦補成分**
產品成分(菌株、添加物、含量)必須以 brief 為準,缺資訊主動標註「需向代理商確認」。

**4. 競品迴避**
影片不可拍到競品,文案不可比較「比 X 牌更好」。

**5. 報價隱性成本**
保健品/藥廠案件涉及多輪法務審稿、保護文字字卡剪輯工時、無 BGM 限制等,報價需 ×1.3-1.5 倍。

# 🌐 PART IX:部落格 WordPress 發稿工作流(骨架版)

> **本區性質**:🟡 骨架版,**短期手動發稿、長期 MCP 連接後補實際後台操作**
> **核心目標**(Ivan 短期需求):**Claude 產的文案能順利進 Amy 部落格,發布前 Amy 只需要人工確認**

## I-1:文章結構模板

### 標準結構
```
1. H1 標題(3 個備選):SEO 優先 / 雜誌感 / 社群擴散
2. 開場情境(地址 + 第一印象)
3. 空間描寫
4. 品項深度描寫(H3 拆解)
5. Amy 私房筆記(≥300 字)
   ├─ 點餐建議
   ├─ 搭配提案
   ├─ 到訪時間
   ├─ 適合場合
   └─ 可跳過的(只在客人自選菜時才寫)
6. 店家資訊
7. 訂位交通
```

### 字數
- **總字數**:1400 字以上
- **Amy 私房筆記**:≥300 字

## I-2:SEO 欄位清單

每篇必填:

| 欄位 | 規格 |
|------|------|
| **H1** | 含主關鍵字,60 字內 |
| **Meta Description** | 160 字內 |
| **Slug** | 英文小寫,以連字號分隔(`-`) |
| **Alt Text** | 每張圖必填,含關鍵字 |
| **內部連結** | ≥ 2 個(連到 Amy 過去相關文章) |
| **主關鍵字** | 1 個 |
| **長尾關鍵字** | 3 個 |
| **Hashtag** | 1 個品牌錨點(`#amy親子`)+ 3-4 個依主題 |
| **特色圖片** | 主角菜高質感橫式圖 |

## I-3:E-E-A-T 規範

Google 部落格 SEO 核心:

| 維度 | 怎麼做 |
|------|-------|
| **Experience**(實際體驗) | 寫實際體驗細節(口感、份量、價格、停車狀況) |
| **Expertise**(專業知識) | 食材知識、品類知識、業界背景 |
| **Authoritativeness**(權威) | 一致發布、累積同主題文章、建立利基 |
| **Trustworthiness**(可信) | **不腦補**、實際吃過才寫過程、廠商素材如實標註 |

## I-4:格式規範

- **Markdown 包進 code block** → 方便 Amy 複製貼進 WordPress 後台
- **H2/H3 層級正確**(不跳級)
- **不用全形括號**:`(永和)` 而非 `(永和)`(SEO 工具識別困難)
- **數字用半形** + `$` 符號

## I-5:短期 vs 長期工作流

### 🟢 短期(現在):Claude 產出「WordPress 就緒格式」

**目標**:Amy 複製貼上後**直接就有正確格式**,不用手動調。

**做法**:
- 內文用 Markdown,但用 code block 包覆
- SEO 欄位**全部列在文末**,Amy 複製貼上對應 WP 欄位
- 圖片位置用文字標記(「→ 此處放主角菜照片」),Amy 手動上傳

### 🟡 中期(未來):Claude 透過 Chrome 代理半自動

**目標**:Claude 自動登入 WP 後台、貼文、設定欄位,停在「草稿」狀態。

**前置條件**:
- ☐ Amy 帳號密碼或應用程式密碼設定好
- ☐ 確認登入後台路徑
- ☐ Chrome 代理工具測試通過

### 🔴 長期(以後):WordPress MCP 連接器

**目標**:全自動產文 → 直接草稿 → Amy 一鍵發布。

**前置條件**:
- ☐ WordPress MCP 連接器存在或開發
- ☐ Amy 帳號授權
- ☐ 圖片上傳流程設計(雲端 / 圖庫管理)

## 附錄 2:8 個爆款方向定義

### 1️⃣ 反差懸念型
利用「意料之外」讓人停滑。第一反應「蛤?什麼意思」必須點進去才知道。

### 2️⃣ 內行人切角型
利用「懂的人才懂」的心理優勢。

### 3️⃣ 場景痛點型
直接打中讀者在找什麼。

### 4️⃣ 場景畫面型
利用熱播劇/地標的搜尋熱度。

### 5️⃣ 數字具體型
用數字撐起標題的說服力。

### 6️⃣ 故事感型
用一個小場景開頭,讓人代入。**避免太誇張**(「整桌都安靜了一下」這種戲劇化已被 Ivan 否決)。

### 7️⃣ 比較型
建立「KOL 已經幫你篩選過」的權威感。

### 8️⃣ 情緒共鳴型
直接抓住情緒,看完想 tag 朋友。

→ 8 個方向的標題範例:用 Amy 過往爆款貼文累積後寫入 `voice.md` C 區

## 附錄 3:Reels 開頭三選一鉤子範例

### 模板 A:結果前置型
- 第 1 幀 = 吃完最後一口的滿足表情特寫
- 第 1 幀 = 結帳金額特寫(出乎意料的便宜或貴)
- 第 1 幀 = 整桌菜爆炸畫面

### 模板 B:疑問懸念型
- 大字幕「沒想到 [區域] 這家居然 [賣點]?」+ KOL 驚訝反應
- 大字幕「我不敢相信這個價格...」+ 看到帳單表情

### 模板 C:反差衝擊型
- 「我以為是普通早午餐...」(平凡早餐畫面) → 0.8 秒切「結果端出這個」
- 「以為只是路邊小店...」(店外簡陋) → 0.8 秒切「結果牆上掛米其林推薦」
